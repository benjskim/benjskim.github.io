Project Objective

"For this project, you'll work by yourself to sketch and prototype (using Axure) a way to incorporate weather information into an app. The app itself can be anything: a music player, a trip planner, a sports app... Be creative! You can do anything except an app that only tells the weather. Students will need to make their presentations by recording a [5-min] video."

Ideation, Sketching, and Prototyping

Since this project was meant to be a quick prototyping exercise, I used my previous experiences to ideate an app I'd personally use. I wanted to create a field-finding application that will help users find local courts near their area. Because of the project objective regarding weather information, I figured it'd be great to know when courts are unusable due to the weather. By filtering weather-affected courts, users can plan accordingly on rainy days. This idea led me to start sketching right away:
(Click a picture to enlarge)

The sketching phase is where I start to gain some of my feedback. While my professor liked the idea he wanted more depth to the application, such as the number of people currently present in each court via webcam.

While we agreed a webcam in every single court was unrealistic, it sparked an idea regarding a check-in process for users. The check-in process, in addition to allowing users to better understand the busyness of a certain court, also added depth to the weather portion of this project.

For example, if it rained Monday night but it is sunny and clear on Tuesday morning, then a simple weather app would indicate outdoor courts to be usable. However, let's say for instance the rain left puddles of water in a certain basketball court, rendering it unusable. The app can rely on check-in data from users to intuitively determine if a court is usable. I really enjoyed the depth this idea created for the project, so I started prototyping in Axure:

(Click a picture to enlarge)

Due to the quick nature of the project, the prototype is limited to a single user flow: Find a Court -> Search Results -> Filter -> Weather-Friendly option -> Update. You can also view the profile page for each court, which shows information such as: pictures, daily admission fees (if any), the number of people checked-in, and a Google Maps directions feature.

Future features would include: wider range of sports, user-profile pages, user history, social log-in, and ability to buy admission-passes directly from the app (potential revenue source).

Conclusion

I ended up receiving an A on this project, but not with much feedback unfortunately. Regardless, this project was a fun exercise in formulating a singular user flow and turning it into a viable app.
